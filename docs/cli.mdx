# CLI Commands

StatikAPI ships two CLIs:

- **`create-statikapi`** — scaffolds a new project
- **`statikapi`** — dev and build for your API

---

## `create-statikapi`

Scaffold a new project with templates and deployment presets.

#### With pnpm
```bash
npx create-statikapi my-api
```

#### With yarn
```bash
npx create-statikapi my-api
```

#### With npm
```bash
npx create-statikapi my-api
```

### Flags

- `--template basic|dynamic|remote-data`
- `--yes, -y` — accept defaults (non-interactive)
- `--no-install`
- `--no-gitignore`
- `--package-manager pnpm|npm|yarn`
- `--src-dir <dir>` (default `src-api`)
- `--out-dir <dir>` (default `api-out`)
- `--deploy cloudflare,netlify,github`
- `--help, -h`

**Examples**

```bash
create-statikapi my-api --template dynamic --no-install
```

```bash
create-statikapi my-api --src-dir api --out-dir public/api --deploy cloudflare,github
```

---

## `statikapi dev`

Watches your source and serves an interactive UI.

#### With pnpm
```bash
pnpm dev
```

#### With yarn
```bash
yarn dev
```

#### With npm
```bash
npm run dev
```

**Common flags**

- `--no-ui` — run server without the preview UI
- `--port <number>` — override default dev port (from your bin)
- `--src <dir>` — override source dir (defaults to config or `src-api/`)
- `--out <dir>` — override output dir (defaults to config or `api-out/`)

---

## `statikapi build`

Builds static JSON into `api-out/` (or your configured dir).

#### With pnpm
```bash
pnpm build
```

#### With yarn
```bash
yarn build
```

#### With npm
```bash
npm run build
```

**Useful flags**

- `--pretty` — human-readable `index.json`
- `--src <dir>` — override source dir
- `--out <dir>` — override output dir
- `--clean` — remove old output before writing

**Script-only variant**

#### With pnpm
```bash
pnpm run build:api
```

#### With yarn
```bash
yarn build:api
```

#### With npm
```bash
npm run build:api
```

---

## Exit codes

- `0` success
- non-zero indicates an error (parsing, user module errors, etc.)
