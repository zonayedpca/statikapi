The core CLI provides two commands: **`dev`** and **`build`**.

---

## `statikapi dev`

Watches your source and serves an interactive UI at `/_ui`.

> **Note:** The default **source dir** is `src-api/` and **output dir** is `api-out/`, both configurable via your [Configuration](../config). You can also override them via CLI flags `--src` and `--out`.

<Tabs group="dev">
  <Tab label="pnpm">

```bash
pnpm dev
```

  </Tab>
  <Tab label="yarn">

```bash
yarn dev
```

  </Tab>
  <Tab label="npm">

```bash
npm run dev
```

  </Tab>
</Tabs>

**Common flags**

- `--no-ui` â€” run server without the preview UI
- `--port <number>` â€” override dev port
- `--src <dir>` â€” override source dir (defaults to config or `src-api/`)
- `--out <dir>` â€” override output dir (defaults to config or `api-out/`)

---

## `statikapi build`

Builds static JSON into your output directory (default `api-out/`, configurable via [Configuration](../config)).

By default, all JSON files are **minified** â€” optimized for hosting and CI/CD pipelines.  
If you prefer readable output, pass the `--pretty` flag.

**Useful flags**

- `--pretty` â€” output human-readable JSON
- `--src <dir>` â€” override source dir (default `src-api/`)
- `--out <dir>` â€” override output dir (default `api-out/`)
- `--clean` â€” remove old output before writing
